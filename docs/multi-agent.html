Multi-Agent

A single agent handles one task at a time. Multi-agent lets you run several specialized agents inside your app — each focused on one thing — and combine their results. All agents run in the same process on the same device.

Why Multi-Agent?

A single agent handles one task at a time with one context window. Multi-agent systems let you:

  • Specialize — give each agent a focused system prompt and only the tools it needs
  • Parallelize — run independent subtasks at the same time, on device
  • Compose — chain the output of one agent into the input of another
  • Bridge — delegate heavy work from your on-device agent to a backend agent your team hosts

Coordination Models

There are three ways to coordinate agents, each suited to a different kind of problem:

Graph DAG

Parallel analysis in the same app
  • Workflow is known up front
  • Independent nodes run in parallel
  • Dependent nodes wait automatically
  • Run multiple focused agents on the same data simultaneously

Swarm Dynamic

In-app routing to the right specialist
  • Routing decided by the model
  • A coordinator agent hands off to whichever specialist fits
  • Handles open-ended user requests
  • Flexible but less predictable

A2A HTTP

On-device + backend
  • Your app's local agent delegates heavy work to a cloud agent your team hosts
  • Remote agent looks like a local tool
  • Backend can be Python or TypeScript Strands SDK
  • Private tasks stay on device; intensive tasks go to the backend

Choosing a Coordination Model

QuestionAnswer → Use
Do I know all the steps in advance?Yes → Graph
Should some steps run in parallel on device?Yes → Graph
Does routing depend on what the user actually asked?Yes → Swarm
Is one task too heavy or private to run on device?Yes → A2A
Does my backend team use Python or TypeScript Strands?Yes → A2A

Agents as Tools

All three models build on the same primitive: an agent can use another agent as a tool. The outer agent calls the inner agent just like any other tool call — it doesn't need to know how the inner agent works.

Writing assistant using a grammar agent as a toolSwift
// A specialized agent that checks grammar and style
let grammarAgent = Agent(
    model: provider,
    systemPrompt: "You are a grammar expert. Review the text and return a concise list of corrections."
)

// Wrap it as a tool the outer agent can call
let grammarTool = AgentTool(
    name: "grammar_check",
    description: "Check grammar and style in a passage of text and return corrections",
    agent: grammarAgent
)

// The writing assistant uses the grammar agent as one of its tools
let writingAssistant = Agent(
    model: provider,
    tools: [grammarTool, suggestTitle],
    systemPrompt: "You help users improve their writing. Use the grammar_check tool to catch errors, then provide overall feedback."
)

let result = try await writingAssistant.run("Review my draft: \(userDraft)")
