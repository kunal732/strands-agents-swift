Multi-Agent

Break complex tasks into pieces, each handled by a specialized agent. Agents coordinate so the combined result is better than any one agent alone.

Why Multi-Agent?

A single agent handles one task at a time with one context window. Multi-agent systems let you:

  • Specialize — give each agent a focused system prompt and only the tools it needs
  • Parallelize — run independent subtasks at the same time
  • Scale — distribute agents across services or machines
  • Compose — chain the output of one agent into the input of another

Coordination Models

There are three ways to coordinate agents, each suited to a different kind of problem:

Graph DAG

Fixed pipeline with parallel steps
  • Workflow is known up front
  • Independent nodes run in parallel
  • Dependent nodes wait automatically
  • Predictable, auditable execution

Swarm Dynamic

Agents hand off to each other at runtime
  • Routing decided by the model
  • Any agent can hand off to another
  • Handles open-ended tasks
  • Flexible but less predictable

A2A HTTP

Agents across process boundaries
  • Agents live in separate services
  • Communicate over HTTP
  • Remote agent looks like a tool
  • Language and platform agnostic

Choosing a Model

QuestionAnswer → Use
Do I know all the steps in advance?Yes → Graph
Should some steps run in parallel?Yes → Graph
Does routing depend on the content of each step?Yes → Swarm
Are some agents in separate services or processes?Yes → A2A
Do I need to call an agent from another language's SDK?Yes → A2A

Agents as Tools

All three models build on the same primitive: an agent can use another agent as a tool. The outer agent calls the inner agent just like any other tool call — it doesn't need to know how the inner agent works.

Inner agent as a toolSwift
// A specialized research agent
let researchAgent = Agent(
    model: provider,
    tools: [searchWeb, fetchPage],
    systemPrompt: "You are a research assistant. Search the web and summarize findings."
)

// Wrap it as a tool the outer agent can call
let researchTool = AgentTool(
    name: "research",
    description: "Research a topic on the web and return a summary",
    agent: researchAgent
)

// Outer agent uses the research agent as a tool
let mainAgent = Agent(
    model: provider,
    tools: [researchTool, writeDocument],
    systemPrompt: "You write detailed reports. Use the research tool to gather information."
)

let result = try await mainAgent.run("Write a report on the history of Swift")
