Agent-to-Agent (A2A)

Agents in different processes or services communicate over HTTP. A remote agent looks exactly like a local tool — callers don't need to know where it lives.

When to Use A2A

  • Agents are deployed as separate services (microservices, lambdas, containers)
  • You want to call a Python or TypeScript Strands agent from Swift
  • You need to reuse an agent across multiple apps or platforms
  • An agent requires different hardware (e.g. a GPU server for image generation)

How It Works

App (Swift) → A2AClient (tool call) → HTTP POST → A2AServer → Remote Agent
The local agent sees A2AClient as a regular tool. The remote agent is unaware it is being called remotely.

Serving an Agent

Wrap any agent in an A2AServer to expose it over HTTP. The server handles request parsing, agent invocation, and response serialization:

research-service/main.swiftSwift
import StrandsAgents

/// Search the web for current information on a topic.
@Tool
func searchWeb(query: String) async throws -> String {
    // your search API call here
    return "Results for: \(query)"
}

let researchAgent = Agent(
    model: try BedrockProvider(config: BedrockConfig(
        modelId: "us.anthropic.claude-sonnet-4-20250514-v1:0"
    )),
    tools: [searchWeb],
    systemPrompt: "You are a research assistant. Search the web and summarize findings concisely."
)

// Expose the agent on port 8080
let server = A2AServer(agent: researchAgent, name: "Research Agent", port: 8080)
try await server.start()
// Server runs at http://localhost:8080

Calling a Remote Agent

Create an A2AClient with the endpoint URL. Pass it to Agent(tools:) — it behaves like any other tool from the caller's perspective:

main-app/main.swiftSwift
import StrandsAgents

// Point at the remote research agent
let remoteResearcher = A2AClient(
    name: "research",
    description: "Search the web and summarize information on any topic",
    endpoint: URL(string: "http://research-service:8080")!
)

// The local agent calls the remote agent exactly like a local tool
let agent = Agent(
    model: try BedrockProvider(config: BedrockConfig(
        modelId: "us.anthropic.claude-sonnet-4-20250514-v1:0"
    )),
    tools: [remoteResearcher, writeDocument],
    systemPrompt: "You write detailed reports. Use the research tool to gather information first."
)

let result = try await agent.run("Write a report on the current state of Swift concurrency")
print(result.output)

Cross-Language Agents

A2A is language-agnostic. A Swift app can call a Python Strands agent, and vice versa. The only requirement is that both sides implement the A2A HTTP protocol:

Calling a Python Strands agent from SwiftSwift
// Python service running at research.internal:8080
// (deployed with the Python Strands SDK's A2A server)
let pythonAgent = A2AClient(
    name: "python_researcher",
    description: "A Python-based research agent with access to specialized databases",
    endpoint: URL(string: "https://research.internal:8080")!
)

let swiftAgent = Agent(model: provider, tools: [pythonAgent])
let result = try await swiftAgent.run("What papers were published on LLM agents in 2024?")

Authentication

Pass a bearer token or other credentials when creating the client. The token is included in every request to the remote agent:

Swift
let client = A2AClient(
    name: "secure_agent",
    description: "An agent behind an auth wall",
    endpoint: URL(string: "https://agents.example.com/researcher")!,
    bearerToken: "eyJhbGciOiJSUzI1NiJ9..."
)

A2A vs. Graph and Swarm

Graph / SwarmA2A
Agent locationSame processAny process or host
CommunicationIn-memory function callsHTTP
LanguagesSwift onlyPython, TypeScript, Swift, any
LatencyMicrosecondsNetwork round-trip
Independent deployNoYes
ScalingSingle processIndependent horizontal scaling
💡

Use A2A with Graph or Swarm — they are not mutually exclusive. A graph node can be an A2A client, and a swarm member can hand off to a remote agent over A2A.
