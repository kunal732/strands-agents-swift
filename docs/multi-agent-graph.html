Graph

Run agents as nodes in a directed acyclic graph (DAG). Independent nodes execute in parallel; dependent nodes wait for their inputs automatically.

When to Use Graph

  • You know all the steps before execution starts
  • Some steps are independent and can run concurrently
  • You want predictable, auditable execution order
  • Downstream agents need the results of upstream agents

How It Works

Each node is an agent with an ID and an optional list of dependencies. The orchestrator runs all nodes whose dependencies are satisfied concurrently, then feeds their outputs to the next nodes.

Basic Example

Research + write pipelineSwift
import StrandsAgents

let researchAgent = Agent(
    model: provider,
    tools: [searchWeb],
    systemPrompt: "You are a research assistant. Find accurate, relevant information."
)

let factCheckAgent = Agent(
    model: provider,
    tools: [searchWeb],
    systemPrompt: "You verify claims. Flag anything unsubstantiated."
)

let writerAgent = Agent(
    model: provider,
    systemPrompt: "You write clear, engaging articles based on provided research."
)

let graph = GraphOrchestrator(nodes: [
    GraphNode(id: "research",    agent: researchAgent),
    GraphNode(id: "fact-check",  agent: factCheckAgent),
    // "write" receives the outputs of both research and fact-check
    GraphNode(id: "write",       agent: writerAgent,
              dependencies: ["research", "fact-check"]),
])

let result = try await graph.run("Write an article about quantum computing")
print(result.output)  // final output from the "write" node

Dependencies

A node's output is automatically forwarded as context to each node that declares it as a dependency. You don't need to wire the outputs manually — the orchestrator handles it.

This four-stage pipeline shows how dependencies chain through the graph:

Multi-stage pipelineSwift
// Stage 1: two independent agents run in parallel
GraphNode(id: "outline",    agent: outlineAgent),
GraphNode(id: "references", agent: referenceAgent),

// Stage 2: draft uses outline + references
GraphNode(id: "draft",  agent: draftAgent,
          dependencies: ["outline", "references"]),

// Stage 3: review uses only the draft
GraphNode(id: "review", agent: reviewAgent,
          dependencies: ["draft"]),

// Stage 4: final polish uses draft + review
GraphNode(id: "polish", agent: polishAgent,
          dependencies: ["draft", "review"])

Parallel Execution

Nodes without dependencies — or whose dependencies have all completed — run concurrently. In the example above, outline and references run at the same time. Execution is ordered by the dependency graph, not by the order you declare the nodes.

💡

Use parallel stages for tasks like summarization, classification, or extraction that do not depend on each other. This can cut wall-clock time significantly on complex pipelines.

Accessing Intermediate Results

The GraphResult includes the outputs of every node, keyed by node ID:

Swift
let result = try await graph.run("Write an article about black holes")

// Final output (from the last node in the graph)
print(result.output)

// Output from a specific intermediate node
print(result.nodeOutputs["research"] ?? "")
print(result.nodeOutputs["fact-check"] ?? "")

Real-World Example: Code Review Pipeline

Three reviewers run in parallel, each focused on a different concern. A summary agent synthesizes all three results.

Parallel code reviewSwift
let securityReviewer = Agent(
    model: provider,
    systemPrompt: "You review code for security vulnerabilities. Be specific."
)
let performanceReviewer = Agent(
    model: provider,
    systemPrompt: "You review code for performance issues. Suggest optimizations."
)
let styleReviewer = Agent(
    model: provider,
    systemPrompt: "You review code for style and readability. Suggest improvements."
)
let summaryWriter = Agent(
    model: provider,
    systemPrompt: "You synthesize multiple code reviews into a single actionable summary."
)

let graph = GraphOrchestrator(nodes: [
    GraphNode(id: "security",    agent: securityReviewer),
    GraphNode(id: "performance", agent: performanceReviewer),
    GraphNode(id: "style",       agent: styleReviewer),
    GraphNode(id: "summary",     agent: summaryWriter,
              dependencies: ["security", "performance", "style"]),
])

let code = """
func fetchUser(id: String) -> User? {
    let query = "SELECT * FROM users WHERE id = '\(id)'"
    return db.execute(query).first
}
"""

let result = try await graph.run("Review this Swift code:\n\n\(code)")
print(result.output)  // synthesized review from all three reviewers
